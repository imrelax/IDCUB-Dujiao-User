# Dujiao-Next User Web · Nuxt 4 深度定制版

这是一个基于 [Dujiao-Next](https://dujiao-next.com) 官方 Vue 3 源码，全面采用 **Nuxt 4 + Vue 3 + Tailwind CSS v4** 进行重构与二次开发的现代化前端项目。

本项目的核心目标是 **大幅提升网站的 SEO 收录能力**、**优化全站响应式 UI/UX 交互体验**，并 **扩展官方尚未实现的自定义功能**，同时保持与旧版系统 API 的完美兼容与路径一致，确保迁移过程中的“零掉站”。

---

## ✨ 核心亮点与技术重构

本项目在保留原有核心业务逻辑的基础上，进行了全方位的技术升级和体验优化：

### 1. 🚀 SSR 服务端渲染与 SEO 优化
- **全站 SSR 架构**：从原有的单页应用 (SPA) 彻底迁移至 Nuxt 4 SSR 架构，搜索引擎蜘蛛可直接抓取完整的 HTML 内容。
- **动态 Sitemap 生成**：自动根据最新商品、分类、文章和教程生成动态站点地图 (`sitemap.xml`)，加速收录。
- **元标签 (Meta Tags) 优化**：全局动态管理 Title、Description、Keywords，并为各个实体页面自动注入相关信息。
- **URL 路径平移**：严格保持与旧版前端完全一致的 URL 路由结构，域名切换时无需配置复杂的 301 重定向，避免权重流失。

### 2. 🎨 现代化的 UI/UX 设计 (Tailwind CSS v4)
- **视觉风格升级**：采用 Tailwind CSS v4 重新设计了各主要模块。商品、分类和公告页面新增了毛玻璃 (Ambient Blur) 的 Hero 头部背景效果。
- **全新登录/注册/找回密码**：重构为现代化的双列布局 (Two-Column Layout)，左侧动态展示网站 Logo 与多语言简介，右侧专注于表单交互，有效降低表单高度，提升转化率。
- **响应式全局页脚 (Footer)**：
  - 动态集成网站 Logo。
  - 响应式多列网格布局，动态加载站点顶级分类。
  - 严格遵守开源协议的版权声明区域（GPLv3 版权平行显示，底部居中展示隐私与服务条款链接）。
- **统一的分页组件**：全站 (博客、公告、教程等) 统一采用重新设计的 `PaginationNav.vue` 分页组件。
- **Trust Bar 信任横幅**：针对虚拟资产 (Telegram、TikTok、X 等账号) 定制的“极速自动发货”、“安全担保交易”信任标签。

### 3. 🌐 全局国际化 (i18n)
- 实现了深度到每一个文案细节的多语言支持（简体中文、繁体中文、英文）。
- 基于用户浏览器偏好、本地存储与 URL 路由，提供精确的语言智能回退与切换能力。

### 4. ⚙️ 高度可定制化 (统一配置入口)
摒弃了分散在代码中的硬编码，所有的扩展功能均统一收口于 `app/config/xiabibi.json` 配置文件中：
- **商品标签与营销徽章**：自定义如“热销”、“秒发”、“爆款”等渐变色标签，并通过 `productIds` 轻松绑定。
- **公告与弹窗系统**：灵活配置首页公告栏和入站全局弹窗。
- **多平台友情链接**：统一配置各大社交平台及友情链接，带独立 Logo 支持。
- **WordPress 教程无缝对接**：通过该配置可开启并指定外部 WordPress 博客站作为“教程中心”的数据源，增强内容营销能力。
- **商品购买须知**：多语言富文本的商品详情页“购买须知”选项卡配置。

### 5. 🔌 极简的反向代理与跨域适配
- **无服务器托管友好**：彻底简化了复杂的反向代理配置，完美适配 Vercel、Cloudflare 等 Serverless (无服务器) 托管平台。
- **开箱即用**：Nuxt 代理自动处理 `/api`、`/uploads`、`/admin` 等路径，对前后端分离部署和跨域问题提供开箱即用的支持。

### 6. 🔄 持续跟进官方更新
- **积极同步**：本项目积极与官方仓库同步更新。
- **无缝衔接**：官方升级的新功能将第一时间迁移并完善，确保前端始终与后端 API 完整适配，实现功能完全兼容。

## 📦 极简安装与无服务器部署

为了兼顾性能与运维成本，本项目极大地简化了安装部署和反向代理流程，深度优化并完美适配 **Vercel**、**Cloudflare** 等无服务器 (Serverless) 托管平台。

### ☁️ 无服务器托管部署 (推荐)

部署时无需修改复杂配置，直接在托管平台导入本 Git 仓库，设置环境变量即可：

**核心环境变量：**
- `VITE_API_BASE_URL` : 您的后端 API 完整地址 (如 `https://api.yourdomain.com`)

只需设置此 API 环境变量地址，前端即可自动接管并完成与后端的通讯，确保 **API 请求**、**附件读取**、**动态 Sitemap** 等各项核心功能完整、稳定地运行。

> 💡 **零代码扩展**：后续如需添加新功能，只需在托管平台添加对应的新环境变量即可生效，**全程无需修改任何代码**，极致简化后期维护成本。

**Vercel 构建配置参考 (自动读取)：**
- Build Command: `npm run build:client && npm run build:server`
- Output Directory: `dist/client`

### 💻 本地开发环境

1. 安装依赖：
   ```bash
   npm install
   ```

2. 运行本地开发服务器：
   ```bash
   npm run dev
   ```

## 🔧 自定义配置指南

项目的核心自定义能力均通过编辑 `app/config/xiabibi.json` 文件实现。在上线前，请务必根据您的业务需求修改该文件：

1. **`productBadges` & `marketingTitles`**：设置商品角标和促销高亮文字。
2. **`friendLinks`**：设置页脚与关于页面的社交平台或友情链接。
3. **`noticeSettings`**：配置首页弹出式公告和滚动公告。
4. **`wordpress`**：配置教程中心的 API 源（对接独立部署的 WordPress 站点，提供丰富的文章内容）。
5. **`purchaseGuide`**：配置商品详情页的购买须知内容。
